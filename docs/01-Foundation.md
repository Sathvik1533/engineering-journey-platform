## 🎯 DAY 7 — Wednesday, 2 July 2026
### MISSION: Async Python — the most skipped topic that matters most for FastAPI.

#### Why This Mission is Prioritized

FastAPI is an asynchronous web framework. Every route you write will be `async def`. Every database query will use `await`. Every external API call will use `async with`. Without understanding async, you can write FastAPI code by copying templates — but you cannot debug it, optimize it, or explain it. This day prevents weeks of confusion during Month 5.

#### Topics & Subtopics

- Why async exists — the blocking I/O problem
- Concurrency vs parallelism — conceptual distinction
- asyncio module — the event loop
- `async def` — defining coroutines
- `await` — suspending execution until coroutine completes
- `asyncio.run()` — entry point for async programs
- `asyncio.gather()` — running multiple coroutines concurrently
- `asyncio.sleep()` — simulating async I/O delays
- async context managers — `async with`
- When NOT to use async — CPU-bound tasks


#### Manual Practice

1. Write sync function: 5 API calls with time.sleep(1) each. Measure total time.
2. Write same as async with asyncio.sleep(1) each. Measure total time. Observe the difference.
3. Use asyncio.gather() to run 5 'API fetch' coroutines simultaneously.
4. Write async function that reads a file using aiofiles (pip install aiofiles)
5. Write simple async 'task queue' — add tasks, run concurrently, collect results

#### Real-World Engineering Connection

When your FastAPI backend handles 100 simultaneous requests, each might query the database (`await db.execute()`), call OpenAI (`await openai.chat.completions.create()`), and read from a vector database (`await qdrant_client.search()`). Without async, request 2 waits for request 1 to finish all three operations. With async, all 100 requests interleave efficiently. This is why FastAPI is 10x faster than Flask for I/O-heavy applications.
