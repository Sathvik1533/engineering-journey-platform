## The Five Engineering Laws
1. Comfortable implementation is the only gate: Move forward when you can build, explain, modify, and debug independently, not when you have perfected it.
2. Every technology must unlock something: Python -> Backend logic; SQL -> DB integration; HTML/CSS -> UI; React -> Full Stack UI; FastAPI -> API services; Docker -> Deployment.
3. Build before moving forward: A phase is complete only when code is built and committed, not when videos are watched.
4. AI is a reviewer, not the developer: 
   - Stage 1 (Buffer/Python): Zero AI. Write ALL code manually.
   - Stage 2 (SQL/Frontend): AI provides hints when stuck.
   - Stage 3 (FastAPI/Projects): AI reviews code after you write it.
   - Stage 4 (AI Integration): Pair-program.
5. Consistency over intensity: 3-4 focused hours daily beats 10-hour weekend sprints.
